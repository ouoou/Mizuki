## What - 是什么

### 🎯 定义
Java 线程有**六种状态**，定义在 `Thread.State` 枚举中。这些状态描述了线程在其生命周期中的不同阶段，以及线程在不同状态之间的转换条件。

简单来说，线程状态是线程在其生命周期中的**不同阶段**，从创建到销毁的完整过程。

### 🔍 本质特征
- **六种状态**：NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED
- **状态转换**：状态之间可以相互转换，但有特定的转换条件
- **不可逆性**：某些状态转换是不可逆的（如 TERMINATED）
- **操作系统映射**：Java 线程状态映射到操作系统的线程状态

### 📐 基本结构/组成
```
Java 线程状态枚举：
public enum State {
    NEW,              // 新建
    RUNNABLE,          // 可运行
    BLOCKED,           // 阻塞
    WAITING,           // 等待
    TIMED_WAITING,     // 定时等待
    TERMINATED         // 终止
}
```

### 📝 关键术语
| 术语                       | 说明                         |
| ------------------------ | -------------------------- |
| **NEW（新建）**               | 线程被创建但尚未启动的状态              |
| **RUNNABLE（可运行）**        | 线程正在 JVM 中执行或准备执行的状态       |
| **BLOCKED（阻塞）**           | 线程等待获取监视器锁的状态              |
| **WAITING（等待）**           | 线程无限期等待其他线程执行特定操作的状态     |
| **TIMED_WAITING（定时等待）**  | 线程在指定时间内等待的状态              |
| **TERMINATED（终止）**        | 线程执行完成或异常终止的状态             |
| **状态转换（State Transition）** | 线程从一个状态转换到另一个状态的过程        |
| **线程生命周期（Thread Lifecycle）** | 线程从创建到销毁的完整过程            |

### ❓ 为什么没有 RUNNING 状态？
在 Java 的 `Thread.State` 中没有单独的 RUNNING 状态，原因是：
1. **RUNNABLE 已包含两种子状态**：就绪（Ready）和运行中（Running）都归为 RUNNABLE。
2. **调度由操作系统决定**：是否正在 CPU 上运行属于操作系统层面的瞬时状态，JVM 不区分。
3. **查询结果具有时效性**：`getState()` 可能在查询瞬间就绪或运行，分不出稳定的“正在运行”状态。
4. **简化模型**：合并就绪/运行，减少状态数量，便于跨平台实现。

## Why - 为什么

### 🧠 设计原理
线程状态的设计基于以下原理：

1. **生命周期管理**：
   - 需要清晰定义线程的各个阶段
   - 便于操作系统和 JVM 管理线程
   - 提供线程状态的查询接口

2. **调度决策**：
   - 操作系统需要根据线程状态进行调度
   - 不同状态的线程有不同的调度策略
   - 阻塞状态的线程不参与调度

3. **调试和监控**：
   - 开发者需要了解线程的当前状态
   - 便于诊断线程问题（死锁、阻塞等）
   - 提供线程转储（Thread Dump）信息

### 💡 解决的问题
- **问题1**: **线程生命周期管理**
  - 需要明确线程在什么时候处于什么状态
  - 便于系统管理和资源回收
  
- **问题2**: **调度优化**
  - 操作系统可以根据线程状态优化调度
  - 阻塞的线程不占用 CPU 资源
  
- **问题3**: **问题诊断**
  - 可以通过线程状态诊断死锁、阻塞等问题
  - 提供调试和监控的基础

### ⚡ 优势/优点
1. **清晰的生命周期**：明确定义线程的各个阶段
2. **便于管理**：操作系统和 JVM 可以根据状态管理线程
3. **便于调试**：可以通过状态诊断问题
4. **标准化**：统一的线程状态定义，便于理解

### ⚠️ 局限性/缺点
- **状态粒度**：某些状态可能不够细粒度
- **平台差异**：不同平台的状态实现可能有差异
- **状态查询开销**：频繁查询线程状态有一定开销

### 🔬 为什么这样设计
线程状态采用六种状态的设计：

1. **覆盖完整生命周期**：从创建到销毁的所有阶段
2. **区分不同阻塞原因**：BLOCKED、WAITING、TIMED_WAITING 区分不同的阻塞场景
3. **映射操作系统状态**：与操作系统的线程状态对应
4. **便于实现**：状态数量适中，不会过于复杂

## How - 如何使用

### 🔧 基本步骤
理解和使用线程状态的基本步骤：

1. **了解六种状态**:
   - 理解每种状态的含义和特征
   - 知道什么情况下线程处于什么状态

2. **理解状态转换**:
   - 理解状态之间的转换条件
   - 知道哪些转换是可能的，哪些是不可能的

3. **查询线程状态**:
   - 使用 `Thread.getState()` 查询线程状态
   - 使用 jstack 查看线程转储

4. **根据状态诊断问题**:
   - 分析线程状态，找出问题
   - 根据状态优化程序

### 💻 代码实现

#### 查询线程状态
```java
public class ThreadStateExample {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        // NEW 状态
        System.out.println("After creation: " + thread.getState()); // NEW
        
        thread.start();
        
        // RUNNABLE 状态
        System.out.println("After start: " + thread.getState()); // RUNNABLE
        
        Thread.sleep(100);
        
        // TIMED_WAITING 状态（sleep）
        System.out.println("During sleep: " + thread.getState()); // TIMED_WAITING
        
        thread.join();
        
        // TERMINATED 状态
        System.out.println("After completion: " + thread.getState()); // TERMINATED
    }
}
```

#### 演示各种状态
```java
public class AllThreadStates {
    public static void main(String[] args) throws InterruptedException {
        // 1. NEW 状态
        Thread thread1 = new Thread(() -> {});
        System.out.println("NEW: " + thread1.getState());
        
        // 2. RUNNABLE 状态
        Thread thread2 = new Thread(() -> {
            while (true) {
                // 运行中
            }
        });
        thread2.start();
        Thread.sleep(10);
        System.out.println("RUNNABLE: " + thread2.getState());
        thread2.interrupt();
        
        // 3. BLOCKED 状态
        Object lock = new Object();
        Thread thread3 = new Thread(() -> {
            synchronized (lock) {
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        thread3.start();
        Thread.sleep(10);
        
        Thread thread4 = new Thread(() -> {
            synchronized (lock) {
                // 等待获取锁
            }
        });
        thread4.start();
        Thread.sleep(10);
        System.out.println("BLOCKED: " + thread4.getState());
        
        // 4. WAITING 状态
        Object waitLock = new Object();
        Thread thread5 = new Thread(() -> {
            synchronized (waitLock) {
                try {
                    waitLock.wait(); // 无限期等待
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        thread5.start();
        Thread.sleep(10);
        System.out.println("WAITING: " + thread5.getState());
        synchronized (waitLock) {
            waitLock.notify();
        }
        
        // 5. TIMED_WAITING 状态
        Thread thread6 = new Thread(() -> {
            try {
                Thread.sleep(1000); // 定时等待
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        thread6.start();
        Thread.sleep(10);
        System.out.println("TIMED_WAITING: " + thread6.getState());
        
        // 6. TERMINATED 状态
        Thread thread7 = new Thread(() -> {});
        thread7.start();
        thread7.join();
        System.out.println("TERMINATED: " + thread7.getState());
    }
}
```

#### 状态转换示例
```java
public class StateTransition {
    public static void main(String[] args) throws InterruptedException {
        Object lock = new Object();
        
        Thread thread = new Thread(() -> {
            System.out.println("1. State after creation: " + Thread.currentThread().getState());
            
            synchronized (lock) {
                System.out.println("2. State after acquiring lock: " + Thread.currentThread().getState());
                
                try {
                    System.out.println("3. Before wait: " + Thread.currentThread().getState());
                    lock.wait(1000); // TIMED_WAITING
                    System.out.println("4. After wait: " + Thread.currentThread().getState());
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        
        System.out.println("0. Initial state: " + thread.getState()); // NEW
        
        thread.start();
        Thread.sleep(50);
        System.out.println("Started: " + thread.getState()); // RUNNABLE
        
        Thread.sleep(100);
        System.out.println("During wait: " + thread.getState()); // TIMED_WAITING
        
        thread.join();
        System.out.println("Final state: " + thread.getState()); // TERMINATED
    }
}
```

### 🧩 关键步骤详解
- **查询线程状态的关键点**: 
  - 使用 `Thread.getState()` 方法查询
  - 状态是瞬时的，可能查询时已经改变
  - 需要结合线程转储（jstack）分析
  
- **理解状态转换的关键点**:
  - 状态转换有特定的条件
  - 某些转换是不可逆的（如 TERMINATED）
  - 状态转换可能失败（如 interrupt）

- **诊断问题的关键点**:
  - BLOCKED 状态可能表示死锁
  - WAITING 状态可能表示线程在等待条件
  - 大量线程处于同一状态可能表示性能问题

### 📈 复杂度分析
- **时间复杂度**: 
  - **查询状态**：O(1)，直接读取线程状态字段
  - **状态转换**：O(1)，但可能涉及系统调用
  
- **空间复杂度**: 
  - **状态存储**：O(1)，每个线程一个状态字段
  
- **性能分析**: 
  - **状态查询**：开销很小，可以频繁查询
  - **状态转换**：某些转换涉及系统调用，有一定开销

## 原理全流程解析

### 🔄 完整流程概述
线程状态转换的完整流程：线程从 NEW 状态开始，通过 start() 进入 RUNNABLE 状态。在 RUNNABLE 状态下，线程可能因为获取锁失败进入 BLOCKED 状态，或因为 wait()、sleep() 等操作进入 WAITING/TIMED_WAITING 状态。当线程执行完成或异常终止时，进入 TERMINATED 状态。整个过程由 JVM 和操作系统协同管理。

### 📋 详细流程步骤

#### 步骤1: NEW（新建）
**目的**: 线程对象被创建，但尚未启动
**操作**: 
- 使用 `new Thread()` 创建线程对象
- 线程对象已分配内存，但未分配系统资源
- 线程尚未开始执行

**结果**: 线程处于 NEW 状态
**关键点**: 
- 只有创建了线程对象，还没有调用 start()
- 此时线程不占用 CPU 资源
- 可以设置线程属性（名称、优先级等）

#### 步骤2: RUNNABLE（可运行）
**目的**: 线程可以执行，等待 CPU 调度
**操作**: 
- 调用 `thread.start()` 启动线程
- JVM 请求操作系统创建线程
- 线程进入就绪队列，等待 CPU 调度

**结果**: 线程处于 RUNNABLE 状态
**关键点**: 
- RUNNABLE 包括"就绪"和"运行"两个子状态
- 线程可能正在执行，也可能在等待 CPU
- 这是线程的主要工作状态

#### 步骤3: BLOCKED（阻塞）
**目的**: 线程等待获取监视器锁
**操作**: 
- 线程尝试获取 synchronized 锁
- 锁被其他线程持有，当前线程阻塞
- 线程进入阻塞队列，等待锁释放

**结果**: 线程处于 BLOCKED 状态
**关键点**: 
- 只有等待 synchronized 锁时才会进入此状态
- 锁释放后，线程会重新进入 RUNNABLE 状态
- 这是竞争锁导致的阻塞

#### 步骤4: WAITING（等待）
**目的**: 线程无限期等待其他线程执行特定操作
**操作**: 
- 调用 `Object.wait()`、`Thread.join()`、`LockSupport.park()`
- 线程释放锁，进入等待状态
- 等待其他线程调用 `notify()`、`notifyAll()` 等

**结果**: 线程处于 WAITING 状态
**关键点**: 
- 需要其他线程的特定操作才能唤醒
- 不会自动唤醒（除非被中断）
- 常用于线程间协调

#### 步骤5: TIMED_WAITING（定时等待）
**目的**: 线程在指定时间内等待
**操作**: 
- 调用 `Thread.sleep()`、`Object.wait(timeout)`、`Thread.join(timeout)`
- 线程进入定时等待状态
- 时间到后自动唤醒，或提前被中断

**结果**: 线程处于 TIMED_WAITING 状态
**关键点**: 
- 有明确的等待时间
- 时间到后自动转换到 RUNNABLE
- 可以被中断提前唤醒

#### 步骤6: TERMINATED（终止）
**目的**: 线程执行完成或异常终止
**操作**: 
- 线程的 run() 方法执行完成
- 线程抛出未捕获的异常
- JVM 清理线程资源

**结果**: 线程处于 TERMINATED 状态
**关键点**: 
- 这是线程的最终状态
- 不能从 TERMINATED 转换到其他状态
- 线程资源会被回收

### 🔗 流程间的关联
- **NEW → RUNNABLE**: 调用 start() 方法
- **RUNNABLE → BLOCKED**: 尝试获取 synchronized 锁失败
- **BLOCKED → RUNNABLE**: 获取到锁
- **RUNNABLE → WAITING**: 调用 wait()、join()、park()
- **WAITING → RUNNABLE**: 被 notify()、notifyAll() 唤醒或被中断
- **RUNNABLE → TIMED_WAITING**: 调用 sleep()、wait(timeout)、join(timeout)
- **TIMED_WAITING → RUNNABLE**: 时间到或被中断
- **RUNNABLE → TERMINATED**: run() 方法执行完成或异常

### ⚙️ 关键机制说明
- **状态管理机制**: 
  - JVM 维护每个线程的状态
  - 状态转换由 JVM 和操作系统协同完成
  - 状态查询通过 Thread.getState() 实现

- **阻塞机制**: 
  - BLOCKED 是竞争锁导致的阻塞
  - WAITING/TIMED_WAITING 是主动等待
  - 阻塞的线程不占用 CPU 资源

- **唤醒机制**: 
  - notify()/notifyAll() 唤醒 WAITING 状态的线程
  - 锁释放唤醒 BLOCKED 状态的线程
  - 时间到或中断唤醒 TIMED_WAITING 状态的线程

### 🎯 流程优化点
- **减少状态转换**: 
  - 减少不必要的阻塞和等待
  - 优化锁竞争，减少 BLOCKED 状态
  - 合理使用超时，避免无限等待

- **快速状态恢复**: 
  - 及时释放锁，让 BLOCKED 线程快速恢复
  - 及时唤醒等待的线程
  - 避免线程长时间处于阻塞状态

```
┌─────────┐
│  NEW    │ ──> 线程对象（未启动）
└────┬────┘
     │
     │ start()
     ▼
┌─────────┐
│RUNNABLE │ ──> 线程执行中/等待CPU
└────┬────┘
     │
     ├──[锁竞争]──> BLOCKED ──> 等待锁释放
     ├──[wait()]──> WAITING ──> 等待通知
     └──[sleep()]──> TIMED_WAITING ──> 等待时间到
     │
     │ run()完成
     ▼
┌─────────┐
│TERMINATED│ ──> 线程结束
└─────────┘
```

### 🎨 时序图

#### 线程状态转换时序
```
时间轴 ──────────────────────────────────>
线程:  NEW ──> RUNNABLE ──> BLOCKED ──> RUNNABLE ──> WAITING ──> RUNNABLE ──> TERMINATED
       │         │           │           │           │           │           │
       │         │           │           │           │           │           │
      创建      启动        获取锁失败    获取到锁     wait()    notify()   完成

说明：
- 状态转换是瞬时的
- 线程在 RUNNABLE 状态可能被多次调度
- 阻塞和等待状态不占用 CPU
```

### 📐 层次结构图

#### 状态分类
```
线程状态
│
├── 未启动
│   └── NEW
│
├── 可执行
│   └── RUNNABLE
│       ├── 就绪（等待CPU）
│       └── 运行（正在执行）
│
├── 阻塞类
│   ├── BLOCKED（等待锁）
│   ├── WAITING（无限期等待）
│   └── TIMED_WAITING（定时等待）
│
└── 已结束
    └── TERMINATED
```

### 🔍 关键节点详解
- **NEW → RUNNABLE 节点**: 
  - 这是线程生命周期的起点
  - 只能通过 start() 转换
  - 转换后线程开始执行

- **RUNNABLE → BLOCKED 节点**: 
  - 竞争锁时的状态转换
  - 表示线程在等待获取锁
  - 锁释放后自动恢复

- **RUNNABLE → WAITING/TIMED_WAITING 节点**: 
  - 主动等待的状态转换
  - 需要其他线程的配合才能恢复
  - 常用于线程间协调

- **任何状态 → TERMINATED 节点**: 
  - 线程的最终状态
  - 不可逆的转换
  - 线程资源会被回收

### 💡 流程图解读要点
1. **起点和终点**: 
   - **起点**：NEW 状态（线程创建）
   - **终点**：TERMINATED 状态（线程结束）

2. **关键路径**: 
   - NEW → RUNNABLE → (BLOCKED/WAITING/TIMED_WAITING) → RUNNABLE → TERMINATED

3. **分支条件**: 
   - 锁竞争 → BLOCKED
   - wait()/join() → WAITING
   - sleep()/wait(timeout) → TIMED_WAITING

4. **异常处理**: 
   - 线程异常时直接进入 TERMINATED
   - 中断可以提前唤醒等待的线程

## 核心要点

### 🔑 核心思想
> **Java 线程有六种状态：NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED。状态之间可以相互转换，但需要满足特定条件。理解线程状态有助于调试多线程问题和优化程序性能。**

### 🎯 关键技巧
1. **记忆六种状态**：NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED
2. **理解转换条件**：每种状态转换都有特定的触发条件
3. **区分阻塞类型**：BLOCKED 是竞争锁，WAITING/TIMED_WAITING 是主动等待
4. **使用工具诊断**：使用 jstack 查看线程状态，诊断问题

### ⚠️ 易错点
| 易错点 | 错误示例 | 正确做法 | 原因 |
|--------|----------|----------|------|
| 混淆 RUNNABLE 子状态 | 认为 RUNNABLE 只有运行状态 | 理解 RUNNABLE 包括就绪和运行 | Java 不区分就绪和运行 |
| 混淆 BLOCKED 和 WAITING | 认为所有阻塞都是 BLOCKED | 区分竞争锁和主动等待 | 阻塞原因不同 |
| 认为状态转换可逆 | 认为可以从 TERMINATED 恢复 | 理解 TERMINATED 是最终状态 | 线程已结束 |
| 忽略状态查询时机 | 在状态转换前查询 | 在适当时机查询状态 | 状态是瞬时的 |

## 常见错误

### ❌ 错误1: 混淆 RUNNABLE 的子状态
**错误表现**: 认为 RUNNABLE 状态只有"运行"一种情况
**错误原因**: 不理解 Java 的 RUNNABLE 状态包括"就绪"和"运行"两个子状态
**正确做法**: 理解 RUNNABLE 状态包括等待 CPU 调度和执行中两种情况
**示例**:
```java
// ❌ 错误理解：认为 RUNNABLE 就是正在运行
public class WrongUnderstanding {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            // 错误：认为这里一定是 RUNNABLE 且正在运行
            System.out.println(Thread.currentThread().getState()); // RUNNABLE
        });
        thread.start();
        // 实际上：RUNNABLE 包括就绪（等待CPU）和运行（正在执行）
    }
}

// ✅ 正确理解：RUNNABLE 包括就绪和运行
public class CorrectUnderstanding {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            // RUNNABLE 状态可能表示：
            // 1. 正在 CPU 上执行（运行）
            // 2. 在就绪队列中等待 CPU（就绪）
            System.out.println(Thread.currentThread().getState()); // RUNNABLE
        });
        thread.start();
    }
}
```

### ❌ 错误2: 混淆 BLOCKED 和 WAITING
**错误表现**: 认为所有阻塞都是 BLOCKED 状态
**错误原因**: 不理解 BLOCKED 和 WAITING/TIMED_WAITING 的区别
**正确做法**: 理解 BLOCKED 是竞争锁导致的，WAITING/TIMED_WAITING 是主动等待
**示例**:
```java
// ❌ 错误理解：认为所有阻塞都是 BLOCKED
public class WrongBlocking {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                Thread.sleep(1000); // 错误：认为这是 BLOCKED
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        thread.start();
        Thread.sleep(10);
        System.out.println(thread.getState()); // 实际是 TIMED_WAITING，不是 BLOCKED
    }
}

// ✅ 正确理解：区分不同类型的阻塞
public class CorrectBlocking {
    public static void main(String[] args) throws InterruptedException {
        Object lock = new Object();
        
        // BLOCKED：竞争锁导致的阻塞
        Thread t1 = new Thread(() -> {
            synchronized (lock) {
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        t1.start();
        Thread.sleep(10);
        
        Thread t2 = new Thread(() -> {
            synchronized (lock) { // 等待获取锁，BLOCKED
            }
        });
        t2.start();
        Thread.sleep(10);
        System.out.println("BLOCKED: " + t2.getState()); // BLOCKED
        
        // WAITING：主动等待
        Object waitLock = new Object();
        Thread t3 = new Thread(() -> {
            synchronized (waitLock) {
                try {
                    waitLock.wait(); // WAITING
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        t3.start();
        Thread.sleep(10);
        System.out.println("WAITING: " + t3.getState()); // WAITING
        
        // TIMED_WAITING：定时等待
        Thread t4 = new Thread(() -> {
            try {
                Thread.sleep(1000); // TIMED_WAITING
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        t4.start();
        Thread.sleep(10);
        System.out.println("TIMED_WAITING: " + t4.getState()); // TIMED_WAITING
    }
}
```

### ❌ 错误3: 认为可以从 TERMINATED 恢复
**错误表现**: 认为线程在 TERMINATED 状态后可以重新启动
**错误原因**: 不理解 TERMINATED 是最终状态，不可逆
**正确做法**: 理解 TERMINATED 状态的线程不能重新启动，需要创建新线程
**示例**:
```java
// ❌ 错误做法：尝试重新启动已终止的线程
public class WrongRestart {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            System.out.println("Thread running");
        });
        
        thread.start();
        thread.join();
        
        System.out.println("State: " + thread.getState()); // TERMINATED
        
        // 错误：不能重新启动已终止的线程
        // thread.start(); // 会抛出 IllegalThreadStateException
    }
}

// ✅ 正确做法：创建新线程
public class CorrectRestart {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            System.out.println("Thread running");
        });
        
        thread.start();
        thread.join();
        
        System.out.println("State: " + thread.getState()); // TERMINATED
        
        // 正确：创建新线程
        Thread newThread = new Thread(() -> {
            System.out.println("New thread running");
        });
        newThread.start();
    }
}
```

## 记忆技巧

### 🧠 记忆口诀
> **"新建启动可运行，锁竞争时阻塞，主动等待分两种，定时等待有期限，执行完成就终止"**

解释：
- **新建启动可运行**：NEW → start() → RUNNABLE
- **锁竞争时阻塞**：竞争 synchronized 锁 → BLOCKED
- **主动等待分两种**：WAITING（无限期）和 TIMED_WAITING（定时）
- **定时等待有期限**：TIMED_WAITING 有明确的等待时间
- **执行完成就终止**：run() 完成 → TERMINATED

### 📝 思维导图/流程图
```
线程状态记忆树：
线程状态
│
├── 未启动
│   └── NEW（新建）
│
├── 可执行
│   └── RUNNABLE（可运行）
│       ├── 就绪（等待CPU）
│       └── 运行（正在执行）
│
├── 阻塞类
│   ├── BLOCKED（阻塞）- 竞争锁
│   ├── WAITING（等待）- 无限期等待
│   └── TIMED_WAITING（定时等待）- 有期限等待
│
└── 已结束
    └── TERMINATED（终止）
```

### 🔗 类比理解
**类比：员工工作状态**

- **NEW** = 新员工入职，但还没开始工作
- **RUNNABLE** = 员工在工作或等待分配任务
- **BLOCKED** = 员工等待使用共享资源（如打印机）
- **WAITING** = 员工等待通知（如等待会议开始）
- **TIMED_WAITING** = 员工等待指定时间（如午休时间）
- **TERMINATED** = 员工离职，不再工作

## 实战案例

### 📚 案例1: 监控线程状态
**问题描述**: 监控应用程序中所有线程的状态，了解系统运行情况
**解决思路**: 
1. 获取所有线程
2. 查询每个线程的状态
3. 统计各状态的线程数量

**代码实现**:
```java
public class ThreadStateMonitor {
    public static void main(String[] args) throws InterruptedException {
        // 创建多个不同状态的线程
        Object lock = new Object();
        
        // RUNNABLE 线程
        Thread runnableThread = new Thread(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                // 运行中
            }
        });
        runnableThread.start();
        
        // BLOCKED 线程
        Thread blockedThread = new Thread(() -> {
            synchronized (lock) {
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        blockedThread.start();
        Thread.sleep(10);
        
        Thread waitingForLock = new Thread(() -> {
            synchronized (lock) { // 等待锁
            }
        });
        waitingForLock.start();
        Thread.sleep(10);
        
        // WAITING 线程
        Object waitLock = new Object();
        Thread waitingThread = new Thread(() -> {
            synchronized (waitLock) {
                try {
                    waitLock.wait();
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        waitingThread.start();
        Thread.sleep(10);
        
        // TIMED_WAITING 线程
        Thread timedWaitingThread = new Thread(() -> {
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        timedWaitingThread.start();
        Thread.sleep(10);
        
        // 监控所有线程状态
        Thread[] allThreads = new Thread[Thread.activeCount()];
        Thread.enumerate(allThreads);
        
        Map<Thread.State, Integer> stateCount = new HashMap<>();
        for (Thread thread : allThreads) {
            if (thread != null) {
                Thread.State state = thread.getState();
                stateCount.put(state, stateCount.getOrDefault(state, 0) + 1);
                System.out.println(thread.getName() + ": " + state);
            }
        }
        
        System.out.println("\n状态统计:");
        stateCount.forEach((state, count) -> 
            System.out.println(state + ": " + count)
        );
    }
}
```

**关键点**: 
- 可以监控所有线程的状态
- 统计各状态的线程数量有助于诊断问题
- 结合线程转储（jstack）可以更详细地分析

### 📚 案例2: 诊断死锁问题
**问题描述**: 通过线程状态诊断死锁问题
**解决思路**: 
1. 创建死锁场景
2. 查询线程状态
3. 分析 BLOCKED 状态的线程

**代码实现**:
```java
public class DeadlockDiagnosis {
    private static final Object lock1 = new Object();
    private static final Object lock2 = new Object();
    
    public static void main(String[] args) throws InterruptedException {
        Thread thread1 = new Thread(() -> {
            synchronized (lock1) {
                System.out.println("Thread1 acquired lock1");
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                synchronized (lock2) {
                    System.out.println("Thread1 acquired lock2");
                }
            }
        });
        
        Thread thread2 = new Thread(() -> {
            synchronized (lock2) {
                System.out.println("Thread2 acquired lock2");
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                synchronized (lock1) {
                    System.out.println("Thread2 acquired lock1");
                }
            }
        });
        
        thread1.start();
        thread2.start();
        
        Thread.sleep(200);
        
        // 诊断线程状态
        System.out.println("Thread1 state: " + thread1.getState()); // BLOCKED
        System.out.println("Thread2 state: " + thread2.getState()); // BLOCKED
        
        // 两个线程都是 BLOCKED 状态，可能存在死锁
        if (thread1.getState() == Thread.State.BLOCKED && 
            thread2.getState() == Thread.State.BLOCKED) {
            System.out.println("Potential deadlock detected!");
        }
    }
}
```

**关键点**: 
- BLOCKED 状态的线程可能表示死锁
- 多个线程同时处于 BLOCKED 状态需要警惕
- 使用 jstack 可以更详细地分析死锁

### 📚 案例3: 线程状态转换追踪
**问题描述**: 追踪线程在整个生命周期中的状态转换
**解决思路**: 
1. 创建线程并监控状态变化
2. 在不同阶段查询状态
3. 记录完整的状态转换过程

**代码实现**:
```java
public class StateTransitionTracking {
    public static void main(String[] args) throws InterruptedException {
        Object lock = new Object();
        List<String> stateHistory = new ArrayList<>();
        
        Thread thread = new Thread(() -> {
            stateHistory.add("In run(): " + Thread.currentThread().getState());
            
            synchronized (lock) {
                stateHistory.add("After acquiring lock: " + Thread.currentThread().getState());
                
                try {
                    stateHistory.add("Before wait: " + Thread.currentThread().getState());
                    lock.wait(1000); // TIMED_WAITING
                    stateHistory.add("After wait: " + Thread.currentThread().getState());
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
            
            stateHistory.add("Before sleep: " + Thread.currentThread().getState());
            try {
                Thread.sleep(100); // TIMED_WAITING
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            stateHistory.add("Final in run(): " + Thread.currentThread().getState());
        });
        
        // 追踪状态转换
        stateHistory.add("After creation: " + thread.getState()); // NEW
        
        thread.start();
        Thread.sleep(10);
        stateHistory.add("After start: " + thread.getState()); // RUNNABLE
        
        Thread.sleep(50);
        stateHistory.add("During wait: " + thread.getState()); // TIMED_WAITING
        
        synchronized (lock) {
            lock.notify();
        }
        
        Thread.sleep(50);
        stateHistory.add("During sleep: " + thread.getState()); // TIMED_WAITING
        
        thread.join();
        stateHistory.add("After join: " + thread.getState()); // TERMINATED
        
        // 打印完整的状态转换历史
        System.out.println("状态转换历史:");
        stateHistory.forEach(System.out::println);
    }
}
```

**关键点**: 
- 可以追踪线程的完整状态转换过程
- 有助于理解线程生命周期
- 可以用于调试和性能分析
